# ImagePet V0.2 手动验收测试指南

本指南详细说明了如何对 ImagePet V0.2 版本进行端到端（E2E）手动功能与交互验收。测试基于 App Sandbox 启用的 macOS 13+ 本地运行环境。

---

## 准备工作

### 1. 编译并运行 App
在终端项目根目录下执行编译并启动脚本：
```bash
./script/build_and_run.sh
```
或者，如果您想观察 App 输出的日志，可以运行：
```bash
./script/build_and_run.sh --logs
```

### 2. 准备测试图片素材
准备一个包含以下类型测试图片的临时文件夹（建议放在桌面）：
- **常规图（5~10张）**：包含 `HEIC`、`PNG`、`JPG/JPEG` 格式。
- **坏图/坏格式（2张）**：一张损坏无法打开的 PNG 或是重命名后缀的假图，以及一张不支持的文件格式（如 `GIF`、`WebP` 或 `PDF`）。

## 验收测试用例

### 场景一：首次启动与输出目录授权 (Sandbox 校验)
**目的**：验证首次启动时是否能正确拦截并引导用户配置合规的安全书签目录。

1. **测试步骤**：
   - 如果是首次运行（或者通过命令行重置了 Defaults），App 启动后应立即自动弹出 macOS 的文件选择器（`NSOpenPanel`），要求选择输出目录。
   - 在选择器中，点击左下角“新建文件夹”，新建一个名为 `ImagePet Output` 的文件夹，然后点击“**Choose Folder**”。
2. **期望结果**：
   - 目录授权成功，App 主界面的底部控制栏中会显示：`Output Folder: /path/to/ImagePet Output`。
   - 界面没有任何黄色警告标志。

---

### 场景二：批量图片压缩核心流程 (Balanced 预设)
**目的**：验证基本拖入、并发排队、即时状态更新及体积节省汇总。

1. **测试步骤**：
   - 保持 Quality 预设为默认的 **Balanced**。
   - 全选准备好的常规图片，拖入 App 主窗口的虚线拖拽区域中；或通过点击“**Add Images**”按钮从文件管理器中全选并导入。
2. **期望结果**：
   - 列表立即列出所有图片，初始状态显示为 `Pending`。
   - 任务以最多 **2 个并发**（`maxConcurrentJobs = 2`）开始执行，执行中的任务显示为 `Processing...`，完成显示为 `Done`。
   - 每张图压缩完成后，右侧立即刷新压缩详情（例如：`3.4 MB -> 820 KB / -75.8%`）。
   - 主窗口左上角的宠物表情在处理中变为 **😋 (nom nom nom...)**，全部完成后变为 **🥳 (Done!)**。
   - 底部汇总栏显示本次批量的总体统计：
     - **Ate**（输入总大小）
     - **Pooped**（输出总大小）
     - **Saved**（保存总体积与比例）
   - 点击“**Reveal in Finder**”按钮，Finder 能够正确打开并高亮选择输出文件夹，其中应该生成了对应的后缀为 `_compressed.jpg` 的图片。

---

### 场景三：坏图拦截与局部失败重试流程
**目的**：验证批处理的鲁棒性，单个文件损坏或不支持不中断整体批次，并可针对失败进行重试。

1. **测试步骤**：
   - 点击主界面底部的“**Compress More**”清除上一次任务。
   - 拖入 3 张图片：1 张正常的 PNG，1 张损坏的 PNG，以及 1 张 GIF。
2. **期望结果**：
   - 正常图顺利通过并完成。
   - 不支持的 GIF 格式立即标红显示：`Failed: Unsupported image format`。
   - 坏图解码失败标红显示：`Failed: Failed to decode image`。
   - 主窗口左上角宠物表情变为 **😵 (Done with issues)**，显示“`1 succeeded, 2 failed`”。
   - 底部控制栏右侧应显示“**Retry Failed**”按钮，点击它，App 将仅重新对那 2 张失败的图发起压缩流程（再次验证失败原因）。

---

### 场景四：桌面宠物悬浮小窗交互
**目的**：验证 Desktop Pet 窗口跨 Space 浮动、自由拖动、关闭、以及表情实时状态联动。

1. **测试步骤**：
   - 点击主界面上的“**Show Pet**”按钮（或使用快捷键 `Cmd + Shift + P`）。
   - 验证桌面右下角是否出现一个小型的宠物悬浮方块。
   - 按住悬浮窗的背景并尝试在屏幕上自由拖拽。
   - 尝试在 Mac 的不同 Spaces（虚拟桌面）之间切换，验证该悬浮窗是否依旧浮动显示在最前（`.floating` 层级）。
   - 再次尝试拖入几张大图进行压缩，观察悬浮窗宠物的表情是否从 **🐡 (Ready)** 切换到 **😋 (Eating)**，最后变成 **🥳 (Done)**。
   - 点击悬浮小窗右上角的圆形叉号“**x**”，或者点击主窗口的“**Hide Pet**”按钮。
2. **期望结果**：
   - 悬浮小窗能够平滑关闭。主窗口的按钮状态自适应变更为“**Show Pet**”。

---

### 场景五：权限失效与防覆盖保护
**目的**：验证安全目录失效防护，以及重名文件写入分配。

1. **测试步骤**：
   - **防覆盖测试**：
     - 不要清空输出目录，再次拖入同名的一张图片进行压缩。
     - 观察输出文件夹内产生的文件名。
     - *期望结果*：不覆盖已有文件，在输出文件夹中生成 `filename-ext_compressed-2.jpg`，以此类推。
   - **权限失效测试**：
     - 关闭 App。
     - 在 Finder 中找到此前选择的输出文件夹 `ImagePet Output`，并将其**重命名**为 `ImagePet Output Changed`。
     - 重新启动 App。
     - *期望结果*：由于书签指向的原始目录被修改或移动，App 会检测到 Sandbox 权限失效，在主界面上提示重新选择文件夹，或者给出黄色警告：`Output folder unavailable`。重新点击 `Choose Folder` 并指派新的目录后可以正常工作。
